# ReadRover Copilot Instructions

## Tech stack (do not deviate)
- **Frontend**: Vite + React + TypeScript
- **Routing**: React Router v6.4+ Data APIs (createBrowserRouter, loaders)
- **Server State**: React Query (@tanstack/react-query) for server state management
- **Forms**: react-hook-form + Zod for forms and validation
- **UI**: Tailwind CSS + shadcn/ui + Radix UI primitives
- **Auth**: httpOnly JWT cookies from FastAPI backend
- **Backend**: FastAPI with SQLAlchemy and asyncpg

## Router rules
- **Always use createBrowserRouter** - never use BrowserRouter or other legacy routers
- **Protect pages via loaders**:
  - `requireAuthLoader` → redirect to `/login` if no `/me` endpoint data
  - `requireGuestLoader` → redirect to `/` if `/me` exists (for login/register pages)
- **Don't run a root meLoader** unless specifically needed - prefer gating on individual protected routes
- **Loader pattern**: Use route loaders for data fetching and authentication checks
- **Navigation**: Use `useNavigate()` and `Link` components, never direct window.location manipulation

## Auth rules
- **Login/register requests are JSON** - never use FormData for auth endpoints
- **Never store tokens in JavaScript** - all tokens must be httpOnly cookies managed by the backend
- **On 401 errors**: 
  1. Clear QueryClient cache using `queryClient.clear()` 
  2. Redirect to `/login?from=...` with current location
  3. Emit unauthorized events via `authEvents.emitUnauthorized()`
- **Use credentials: "include"** in all API requests to include httpOnly cookies
- **Auth state**: Use React Query to cache `/me` endpoint response, never localStorage

## Forms & validation
- **Always use react-hook-form + Zod** - no ad-hoc validators or manual form state
- **Schema-first approach**: Define Zod schemas first, then infer TypeScript types
- **Resolver pattern**: Use `zodResolver(schema)` with useForm
- **Error display**: Show inline errors under inputs using `<FormMessage />` component
- **Submit state**: Disable submit button while `isPending` and show loading indicator
- **Form structure**: Use shadcn Form components (`<Form>`, `<FormField>`, `<FormItem>`, etc.)

## UI rules
- **Tailwind utilities only** - no inline styles except for dynamic positions in canvas/reader components
- **Component library**: Use shadcn/ui components for all primitives:
  - `Button`, `Input`, `Label` for form elements
  - `Dialog`, `Popover`, `DropdownMenu` for overlays
  - `Card`, `Alert` for content containers
- **Theme**: Keep neutral zinc theme throughout - no custom color schemes
- **Highlights**: Use soft yellow utilities for highlights (e.g., `bg-yellow-300/40`)
- **Icons**: Use Lucide React icons consistently
- **Responsive**: Mobile-first approach with Tailwind responsive prefixes

## API patterns
- **Use apiFetch function** for all HTTP requests - never use axios, fetch, or other libraries directly
- **Error handling**: Catch and handle `HttpError` instances properly
- **React Query integration**: 
  - Use `useQuery` for data fetching
  - Use `useMutation` for data modifications
  - Use proper query keys and invalidation patterns
- **Loading states**: Always handle loading, error, and success states in UI

## File organization
- **Features**: Organize by feature in `/src/features/` directory
- **Components**: Shared components in `/src/components/ui/`
- **Pages**: Top-level pages in `/src/pages/`
- **Layouts**: Layout components in `/src/layouts/`
- **Lib**: Utilities and configuration in `/src/lib/`
- **App**: Router and global config in `/src/app/`

## Anti-patterns (reject these)
- ❌ Fetching with axios or raw fetch outside of `apiFetch`
- ❌ localStorage/sessionStorage for tokens or sensitive data
- ❌ Inline styles (except dynamic canvas positions)
- ❌ Manual form validation without Zod
- ❌ Legacy React Router patterns (BrowserRouter, Route components)
- ❌ Storing auth state in React state or localStorage
- ❌ Custom CSS classes - use Tailwind utilities
- ❌ Bypassing the established loader authentication pattern
- ❌ Using any HTTP client other than the configured `apiFetch`
- ❌ Creating new color themes or deviating from zinc base

## Code style
- **TypeScript**: Strict mode, proper type annotations
- **Imports**: Use absolute imports with `@/` prefix
- **Naming**: PascalCase for components, camelCase for functions/variables
- **Exports**: Prefer default exports for pages/layouts, named exports for utilities
- **Async/await**: Use async/await over .then() chains
- **Error boundaries**: Implement proper error handling at route level
